# Contributing

We're glad you're here. Manzanita Research is a small lab, and contributions from the community make everything better.

## Getting started

1. Check the project's README for setup instructions. Most projects use Bun and will have a `bun install` step.
2. Look at open issues. Anything labeled `good first issue` is a reasonable place to start.
3. If you want to work on something bigger, open an issue first so we can talk about it.

## Making changes

- Fork the repo and create a branch.
- Keep changes focused. One pull request, one thing.
- Write tests for anything non-trivial.
- Commit messages: lowercase, concise, describe the *why*. ("fix audio dropout at high buffer sizes" not "Fixed bug")

## Pull requests

- Describe what changed and why.
- If it's a visual change, include a screenshot.
- Be patient with review — we're a small team.

## Reporting bugs

Open an issue. Include:
- What you expected to happen.
- What actually happened.
- Steps to reproduce, if you can.
- Your environment (OS, runtime version, etc.).

## Style

Each project may have its own linting and formatting config. Run the checks before submitting — the CI will catch it either way, but it saves a round trip.

## Code of conduct

We have one. Please read it: [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Questions?

Open an issue or reach out. There are no dumb questions here.
